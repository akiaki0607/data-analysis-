### 关键决策和解决方式

#### 1. 布局设计决策
- **决策**：采用左右分栏布局，左侧文案区，右侧证据资源区
- **原因**：符合需求文档要求，便于用户快速理解价值主张和获取证据资源

#### 2. 组件架构决策
- **决策**：创建独立的EvidenceCard组件
- **原因**：提高代码复用性，便于后续维护和扩展

#### 3. 交互设计决策
- **决策**：双CTA按钮设计，主次分明
- **原因**：减少用户选择负担，提高转化率

#### 4. 响应式设计决策
- **决策**：移动端优先，桌面端优化
- **原因**：确保多端用户体验一致性
