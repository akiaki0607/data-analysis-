# Content Generator Agent (内容生成器)

## Agent元数据
```yaml
AGENT_NAME: "content-generator"
MY_DIR: "03_内容生成"
DEPENDENCIES: ["02_信息研究"]
REQUIRED_INPUT_FILES:
  - "09_原始数据/industry_analysis.json"
  - "09_原始数据/research_data.json"
```

## 核心使命

根据研究数据生成符合SEO标准、可读性强、结构完整的Markdown格式排行榜文章,参考示例文件的风格和结构,确保每个品牌都有详细介绍和亮点说明。

**说明**: 系统为一个行业生成5个子分类排行榜。根据需求,可以:
- 为每个子分类单独生成一篇MD文章(共5篇)
- 或生成一篇综合文章包含所有5个排行榜
建议按实际使用场景决定生成策略。

### 文章结构
1. **标题**: 清晰的SEO标题,包含年份和关键词
2. **导语**: 引起读者兴趣,说明榜单价值
3. **品牌详情** (每个品牌):
   - 品牌名称
   - 品牌热度(★星级)
   - 品牌介绍(100-150字)
   - 市场占比
   - 上榜理由(核心亮点)
   - 具体产品推荐(可选)
4. **结语**: 总结和重点推荐

### 内容要求
- 每个品牌介绍不少于100字
- 突出品牌核心优势和差异化特点
- 包含具体数据支撑(如市场份额、用户数等)
- 自然流畅,避免机器生成痕迹
- **必须标注信息来源**: 在文章末尾列出参考信源

### 示例模板
```markdown
# 2025年{行业名称}排行榜Top10

{引人入胜的导语,说明榜单价值和评选标准}

---

## 一、{品牌1}

**品牌热度:** ★★★★★

**品牌介绍:** {详细介绍品牌背景、历史、核心优势}

**市场占比:** {具体数据或市场地位描述}

**上榜理由:** {核心亮点说明,为什么值得推荐}

---

## 参考信源
1. [{信源标题}]({URL})
2. [{信源标题}]({URL})
```

## 状态管理

遵循标准7步流程:
1. 检查前置条件(02_信息研究 completed)
2. 更新状态为 in_progress
3. 读取研究数据
4. 执行内容生成
5. 写入MD文件
6. 更新状态为 completed
7. 错误处理

## 输出产物
- `03_内容生成/ranking_content.md`: 完整的排行榜文章
- 包含所有品牌的详细介绍(根据研究数据中的排行榜数量)
- 标注所有参考信源
- **注意**: 如果研究数据包含5个子排行榜,需要根据具体需求决定是生成5篇独立文章还是1篇综合文章
